## 💼 Project Overview

헬스케어 플랫폼에서 **병원 예약·진료·결제 시스템**을 설계하고 구축하며, 사용자 경험과 시스템 안정성을 지속적으로 개선해왔습니다.

---

## ⚙️ Back-End Development

### 🏥 예약 및 진료 시스템

#### 📅 예약 서비스 구현
- **Spring Boot** 기반 RESTful API 설계
- 병원별 진료 일정 관리 및 예약 가능 시간 자동 계산
- 예약 충돌 방지를 위한 **낙관적 락** 구현
- 실시간 예약 현황 업데이트 및 알림

#### ⏰ 근무 일정 시간표 집계 로직
- 다수 직원의 근무 시간표를 **병합 알고리즘**으로 통합
- 시간대별 가용 인력 자동 산출
- **Redis 캐싱**으로 조회 성능 최적화

---

### 💳 결제 시스템 고도화

#### 🔄 결제 PG사 교체 작업
- 레거시 PG사에서 신규 PG사로 무중단 전환
- **어댑터 패턴** 적용으로 PG사 변경에 유연한 구조 설계
- 자동 결제, 카카오톡 청구서 등 다양한 결제 수단 지원

#### 🏗️ 결제 서비스 모듈화 (MSA 전환)
- 모놀리식 구조에서 **마이크로서비스 아키텍처**로 분리
- 서비스 간 의존성 제거 및 독립 배포 환경 구축
- **Runnable 패턴** 및 **인터페이스 추상화**로 확장성 확보
- 리소스 비용 절감 및 유지보수성 향상

### 🚀 성능 최적화 및 분석

#### ⚡ Legacy API 성능 개선
- **병원 검색 API 응답 시간 75% 단축** (3.2s → 0.8s)
- **렌더링 시간 76% 개선** (2.1s → 0.5s)
- 데이터 조회 쿼리 분리 및 병렬 처리
- API 분할 전략으로 초기 로딩 최적화

#### 🔍 JMeter를 활용한 부하 테스트
- API 병목 지점 식별 및 부하 원인 분석
- 동시 사용자 증가에 따른 성능 저하 원인 파악
- 쿼리 최적화 및 캐싱 전략으로 처리량 개선

### 🔧 앱 안정성 개선

#### 🔍 백화 현상 원인 분석 및 해결
- 백그라운드 → 포그라운드 전환 시 **화면 백화 현상** 발생
- **`WidgetsBindingObserver`** 구현으로 앱 라이프사이클 모니터링
- `StatefulWidget`으로 전환하여 lifecycle 관리
- **GetX 의존성 주입** 개선으로 `LateInitializationError` 해결
- `permanent: true` 옵션으로 컨트롤러 인스턴스 생명주기 관리
