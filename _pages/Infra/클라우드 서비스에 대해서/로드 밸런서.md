## Load Balancing

로드 밸런싱은 애플리케이션을 지원하는 리소스 풀 전체에 네트워크 트래픽을 균등하게 배포하는 프로세스입니다.

## NLB(Network Load Balancer)

**TCP 로드 밸런서인 NLB(Network Load Balancer)** 는 각 가용 영역에 고정 IP 주소들을 두고 쓸 수 있습니다.  
IP 주소 및 기타 네트워크 정보를 검사하여 트래픽을 최적으로 리디렉션합니다.
이러한 고정 IP 주소들은 변하지 않기 때문에 방화벽 화이트 리스트에 추가될 수 있다.  
이 고정 IP 주소들은 방화벽 화이트 리스트에 추가될 수 있습니다.  
하지만 NLB는 TCP 통신만 허용하기 때문에 HTTPS를 처리하지 않습니다.

## ALB(Application Load Balancer)

애플리케이션 트래픽을 리디렉션하기 위해 HTTP 헤더 또는 SSL 세션 ID와 같은 요청된 콘텐츠를 검사합니다.

### 작동 방식

ALB는 들어오는 트래픽을 EC2 인스턴스를 비롯한 여러 대상에 분산합니다.  
ALB는 이미지와 비디오를 포함하지만 열린 연결을 유지할 필요가 없는 서버에 제품 검색 요청을 전송합니다.

ALB에는 클라이언트의 연결 요청을 확인하는 리스너 구성 요소가 있습니다.  
로드 밸런서가 요청을 등록된 대상으로 라우팅하는 방법을 결정하는 리스너 규칙을 정의할 수 있습니다. 대상 그룹은 등록된 대상을 그룹으로 정렬합니다.  
일반 트래픽을 전체 그룹에 라우팅하는 규칙을 정의할 수 있습니다.

## GLB(Gateway Load Balancer)

- 계층 3과 계층 7에서 작동하며 네트워크 수준에서 밸런싱 및 라우팅 서비스와 게이트웨이 기능을 제공합니다.
- IP 및 인스턴스 대상 유형을 처리합니다.
- 프록시 역할을 하거나 연결을 종료하지 않고, 트래픽을 직접 전달합니다.
- 라우팅 테이블 조회를 사용하여 트래픽을 어디로 라우팅할지 결정합니다.
- 투명한 네트워크 게이트웨이(모든 트래픽의 단일 진입점 및 종료점) 역할을 하며, 수요에 따라 가상 어플라이언스를 확장하면서 트래픽을 분산합니다.
