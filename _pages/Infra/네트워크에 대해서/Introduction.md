## 🌐 컴퓨터 네트워크

컴퓨터와 프린터, 스마트폰과 같은 기타 장치들이 **서로 통신할 수 있도록 연결한 집합**이다.

### 📡 물리적 연결

통신을 위해 컴퓨터와 장치는 **정보 전송을 지원하는 물리적 매체**로 연결되어야 한다.

**연결 매체의 종류:**
- 🔌 **유선 매체**: 구리 케이블, 광섬유
- 📶 **무선 매체**: 무선 주파수 (Wi-Fi, 블루투스 등)

> 💡 컴퓨터는 다양한 네트워크 토폴로지로 매체에 연결될 수 있으며, 사용되는 매체와 상관없이 **통신 프로토콜**이라는 합의된 규칙을 사용한다.

### 🖥️ 네트워크 구성 요소

컴퓨터 네트워크에 포함될 수 있는 요소:
- 💻 개인용 컴퓨터
- 🖧 서버
- 🔧 네트워킹 하드웨어
- ⚙️ 특수/일반 목적의 호스트

### 🏷️ 호스트 식별

네트워크의 각 장치는 다음으로 식별된다:

1. **네트워크 주소** 🔢
   - 통신 프로토콜(예: IP)에 의해 노드를 찾고 식별하는 데 사용
   - 동적으로 변경 가능

2. **호스트명** 📝
   - 노드에 대한 기억하기 쉬운 레이블 역할
   - 초기 할당 후에는 거의 변경되지 않음

## 📦 네트워크 패킷

대부분의 최신 컴퓨터 네트워크는 **패킷 모드 전송**을 기반으로 하는 프로토콜을 사용한다.

> **네트워크 패킷**이란? 교환 네트워크에서 전송되는 형식화된 데이터 단위

### 패킷 구조

패킷은 두 가지 유형의 데이터로 구성된다:

```
┌─────────────┬─────────────────┬─────────────┐
│   Header    │    Payload      │  Trailer    │
│ (제어 정보)  │  (사용자 데이터)  │ (제어 정보)  │
└─────────────┴─────────────────┴─────────────┘
```

#### 1️⃣ 제어 정보 (Header & Trailer)
네트워크가 사용자 데이터를 전달하는 데 필요한 정보:
- 📍 **소스 주소**: 출발지 네트워크 주소
- 🎯 **목적지 주소**: 도착지 네트워크 주소
- 🔍 **오류 감지 코드**: 데이터 무결성 검증
- 🔢 **시퀀싱 정보**: 패킷 순서 관리

#### 2️⃣ 페이로드 (Payload)
실제 전송하려는 사용자 데이터

### ✅ 패킷의 장점

패킷 교환 방식은 회선 교환 방식보다 **전송 매체의 대역폭을 사용자 간에 더 효율적으로 공유**할 수 있다.

## 🏗️ OSI 7 Layer

OSI 모델은 네트워크 통신을 7개의 계층으로 나눈 표준 모델이다.

| 구분 | Layer | 계층명 | PDU | 프로토콜 예시 | 주요 기능 |
|------|-------|--------|-----|--------------|----------|
| 🖥️ **Host Layer** | **7** | Application Layer | Data | HTTP, SMTP, FTP, DNS | 💼 애플리케이션 간 통신 및 서비스 제공 |
| 🖥️ **Host Layer** | **6** | Presentation Layer | Data | ASCII, JPEG, SSL/TLS | 🔄 데이터 변환, 암호화, 압축 |
| 🖥️ **Host Layer** | **5** | Session Layer | Data | NetBIOS, RPC | 🔗 세션 설정, 유지, 종료 관리 |
| 🖥️ **Host Layer** | **4** | Transport Layer | Segment | TCP, UDP | 📨 종단 간 신뢰성 있는 데이터 전송 |
| 📡 **Media Layer** | **3** | Network Layer | Packet | IP, ICMP, OSPF | 🗺️ 라우팅 및 논리적 주소 지정 |
| 📡 **Media Layer** | **2** | Data Link Layer | Frame | Ethernet, Wi-Fi | 🔗 물리적 주소 지정 및 프레임 전송 |
| 📡 **Media Layer** | **1** | Physical Layer | Bit | Cable, Radio | ⚡ 물리적 신호 전송 |

### 계층별 상세 설명

#### 📱 7계층: Application Layer (응용 계층)
- 사용자와 직접 상호작용하는 계층
- 리소스 공유, 원격 파일 접근 등의 고급 서비스 제공

#### 🎨 6계층: Presentation Layer (표현 계층)
- 데이터 형식 변환 (문자 인코딩, 압축, 암호화)
- 시스템 간 데이터 호환성 보장

#### 🤝 5계층: Session Layer (세션 계층)
- 통신 세션 설정, 유지, 종료
- 두 노드 간 대화 제어 및 동기화

#### 🚚 4계층: Transport Layer (전송 계층)
- **신뢰성 있는 데이터 전송** 보장
- 패킷 분할, 재조립, 흐름 제어, 오류 제어
- TCP (연결 지향), UDP (비연결 지향)

#### 🗺️ 3계층: Network Layer (네트워크 계층)
- **라우팅**: 최적 경로 선택
- 논리적 주소 지정 (IP 주소)
- 패킷 전달 및 트래픽 제어

#### 🔗 2계층: Data Link Layer (데이터 링크 계층)
- 물리적으로 연결된 두 노드 간 데이터 전송
- MAC 주소를 이용한 물리적 주소 지정
- 프레임 단위로 데이터 전송

#### ⚡ 1계층: Physical Layer (물리 계층)
- 실제 물리적 매체를 통한 비트 전송
- 전기 신호, 광 신호, 전파 등으로 데이터 표현

## 🔌 네트워크 장비

### 🔄 브리지 및 스위치 (Bridge & Switch)

> **OSI 2계층 (Data Link Layer)** 에서 작동하는 장비

#### 특징 및 차이점

| 구분 | 브리지 | 스위치 |
|------|--------|--------|
| 포트 수 | 2개 | 다수 (8, 16, 24, 48 등) |
| 토폴로지 | 간단한 연결 | 스타 토폴로지 지원 |
| 확장성 | 제한적 | 계단식 연결 가능 |

#### 주요 기능
- ✅ 통신에 관련된 포트로만 프레임 전달
- ✅ 두 개 이상의 네트워크 세그먼트를 연결하여 단일 LAN 형성
- ✅ **충돌 도메인 분할** (네트워크 효율성 향상)
- ✅ 단일 브로드캐스트 도메인 유지

#### 💡 장점
크고 혼잡한 네트워크를 더 작고 효율적인 네트워크로 세분화하여 **전체 네트워크 성능을 개선**한다.

### 🗺️ 라우터 (Router)

> **OSI 3계층 (Network Layer)** 에서 작동하는 장비

#### 주요 기능
라우터는 **패킷의 주소 정보를 처리**하여 네트워크 간 패킷을 전달하는 장치이다.

```
📦 패킷 → 🔍 라우팅 테이블 참조 → 🎯 최적 경로 선택 → 📤 전달
```

#### 라우팅 테이블
- 목적지 네트워크 정보
- 다음 홉(Next Hop) 정보
- 경로 메트릭(비용)
- 네트워크 인터페이스 정보

#### ✅ 주요 역할
- 🌐 서로 다른 네트워크 간 패킷 전달
- 🗺️ 최적 경로 선택 및 관리
- 🚦 트래픽 제어 및 분산
- 🔒 네트워크 보안 경계 설정

## 🛡️ 방화벽 (Firewall)

방화벽은 **네트워크 보안 및 접근 규칙을 제어**하는 장치 또는 소프트웨어이다.

### 배치 위치
```
내부 네트워크 🏢 ←→ 🛡️ 방화벽 ←→ 🌐 외부 네트워크 (인터넷)
```

### 주요 기능

#### 🚫 접근 제어
- ❌ 인식되지 않은 소스의 접근 요청 **차단**
- ✅ 신뢰할 수 있는 소스의 작업 **허용**

#### 🔍 보안 정책
1. **인바운드 규칙**: 외부 → 내부 트래픽 제어
2. **아웃바운드 규칙**: 내부 → 외부 트래픽 제어
3. **포트 및 프로토콜 관리**: 특정 서비스만 허용
4. **IP 필터링**: 특정 IP 주소/대역 차단/허용

### 🎯 중요성

> 🚨 점점 증가하는 사이버 공격 시대에서 방화벽은 **네트워크 보안의 필수 요소**로 자리잡고 있다.

**방화벽의 역할:**
- 🔒 무단 접근 차단
- 🦠 악성 트래픽 필터링
- 📊 네트워크 활동 모니터링
- 🛡️ DDoS 공격 방어

## 🔐 SSL/TLS (보안 통신 프로토콜)

> **SSL (Secure Socket Layer)** / **TLS (Transport Layer Security)**
>
> 인터넷 상에서 안전한 통신을 위한 암호화 프로토콜

### 동작 원리

#### 1️⃣ 인증서 요구
SSL/TLS는 **서버 인증서**를 필요로 한다.

#### 2️⃣ 핸드셰이크 과정
```
1. 클라이언트 → 서버: SSL 연결 요청
2. 서버 → 클라이언트: 인증서 전송 📜
3. 클라이언트: 인증서 검증 🔍
4. 클라이언트: 서버 인증 완료 ✅
5. 양측: 대칭 키 협상 🔑
6. 안전한 통신 시작 🔒
```

#### 3️⃣ 암호화된 통신
협상이 완료되면 클라이언트와 서버는 **매우 안전한 암호화된 터널**에서 통신한다.

### 🔒 보안 특징

| 특징 | 설명 |
|------|------|
| 🔐 **암호화** | 데이터 전송 시 제3자가 내용을 볼 수 없음 |
| ✅ **무결성** | 데이터가 전송 중 변조되지 않았음을 보장 |
| 🆔 **인증** | 통신 상대방의 신원을 확인 |

### 🌐 실제 사용 사례
- **HTTPS**: 웹 브라우징 보안
- **이메일**: SMTP, POP3, IMAP over SSL/TLS
- **VPN**: 가상 사설망 구축
- **금융 거래**: 온라인 뱅킹, 결제 시스템
