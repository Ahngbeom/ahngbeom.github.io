# 2025년 1월 회고록

## 완료한 작업

### 1. 가예약 시스템 마무리 안정화 (2개 이슈) 🔧

12월부터 진행해온 가예약 시스템 안정화 작업을 1월에 최종 완료했습니다.

**이슈:**
- 이슈 #1 (버그) - 가예약 환자 접수 안됨
  - 전화번호를 입력하라는 메시지가 표시되면서 접수가 되지 않는 버그 수정
  - 가예약 환자 정보 입력 유효성 검사 개선

- 이슈 #2 (개선) - 태블릿 가예약 정보 입력 기능 추가
  - 병원에서 빠른 예약으로 입력한 경우 환자 정보가 누락될 수 있는 문제 해결
  - 태블릿 도착 확인 시 환자가 직접 빠진 정보를 입력할 수 있도록 개선
  - 신규 환자 등록 페이지 UI 재활용으로 공수 최소화
  - 신규 환자 등록 → 상세 증상 + 초진문진표 작성 → 약관 동의 → 도착 처리 프로세스 구현

**영향:**
- 가예약 시스템 완전 안정화 완료
- 환자 정보 누락 방지
- 태블릿 셀프 접수 프로세스 구축

### 2. 환자 등록 UX 개선 (2개 이슈) 📝

환자 등록과 진료 신청 시 발생하던 UX 문제를 개선했습니다.

**이슈:**
- 이슈 #3 (개선) - 신규환자 등록 시 이름 띄어쓰기 되지 않음
  - 영어 이름의 경우 이름과 성을 구분하기 어려운 문제 해결
  - 환자명 입력 시 띄어쓰기 허용

- 이슈 #4 (버그) - WEB 진료 신청서 재진 환자 자동 선택 오류
  - 재진 환자가 1명이고 주민등록번호가 유효하지 않을 경우에도 자동 선택되는 버그 수정
  - 주민등록번호 유효성 검사 로직 개선
  - check & disabled 여부 판단 시점을 전체 로직 마지막 단계로 변경

**영향:**
- 환자 등록 입력 편의성 향상
- 외국인 환자 이름 입력 개선
- 진료 신청서 선택 로직 정교화

### 3. DB 작업 (1개 이슈, 진행 중) 🗄️

**이슈:**
- 이슈 #5 (작업) - 사용하지 않는 알람 템플릿 코드 삭제 (진행 중)
  - 미사용 알람 템플릿 15개 삭제 예정
  - 삭제 대상: PAYMENT_SUCCESS, DELIVERY_PAYMENT_REQUEST, FEED_INTEREST_COMMENT 등
  - DB 정리 및 코드 품질 개선

**영향:**
- 미사용 데이터 정리
- DB 구조 간소화

## 주요 성과 및 인사이트

### 1. 가예약 시스템 최종 안정화 완료
2024년 11월 구축 → 12월 안정화 → 2025년 1월 최종 완료:

**해결한 문제:**
- 가예약 환자 접수 오류
- 환자 정보 누락 문제

**결과:**

- 병원에서 안정적으로 사용 가능한 상태

### 2. 환자 등록 UX 정교화
환자 등록 프로세스의 세밀한 개선:
- 외국인 환자 이름 입력 개선 (띄어쓰기 허용)
- 재진 환자 자동 선택 로직 개선
- 주민등록번호 유효성 검사 강화

**영향:**
- 다양한 환자 케이스 대응
- 입력 편의성 향상

### 3. 프론트엔드 비중 증가
커밋 비율이 BE 67.2%, FE 29.5%, Flutter 1.6%, helm 1.6%로 프론트엔드 작업이 크게 증가했습니다:

- 가예약 시스템 안정화 과정에서 FE 작업 증가

### 4. 연속된 안정화 작업의 효과
11월 구축 → 12월 안정화 → 1월 마무리 안정화:

1. **가예약 시스템 안정화 완료**:

2. **환자 등록 UX 세심한 개선**:
   - 외국인 환자 이름 입력 개선
   - 재진 환자 자동 선택 로직 정교화

### 개선이 필요한 점 🔧
1. **이슈 수 감소**:
   - 5개 이슈로 12월(32개) 대비 크게 감소
   - 연말연시 휴가 기간 영향

2. **진행 중인 작업 비율**:
   - 완료율 80%로 12월(96.9%) 대비 낮음
   - 1개 작업이 진행 중

3. **테스트 부족**:
   - 가예약 시스템 E2E 테스트 미구축
   - 환자 등록 테스트 커버리지 부족

### 배운 점 📚
1. **신규 기능 안정화 3단계 프로세스**: 11월 구축 → 12월 안정화 → 1월 마무리 안정화 패턴이 효과적입니다. 신규 기능은 최소 3개월의 안정화 기간이 필요합니다.

2. **UX 세심함의 중요성**: 외국인 환자 이름 띄어쓰기, 재진 환자 자동 선택 등 세밀한 UX 개선이 전체 사용자 경험을 크게 향상시킵니다.

3. **연말연시 생산성 영향**: 연말연시 기간에는 이슈 수가 크게 감소하지만, 커밋 수는 유지되어 개발 활동은 지속되었습니다.

4. **프론트엔드 비중 증가**: 가예약 시스템 안정화 과정에서 FE 작업이 증가하며 BE/FE 균형이 더 좋아졌습니다. DB Migration 7개 커밋 포함 총 68개.
