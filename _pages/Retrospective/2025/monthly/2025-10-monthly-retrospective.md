## 주요 성과

### 1. 운영 서버 안정성 개선 (이슈 #1, 이슈 #2)
- 힙 메모리 누수 현상을 서버/프론트 양측에서 분석하고 해결
- Factory 객체 프로세스 개선을 통한 메모리 관리 최적화
- 당일 진료 조회 API 메모리 최적화

### 2. 대규모 API 리팩토링 (이슈 #3)
- 태블릿/QR/병홈 예약 진료 신청서 API 전면 리팩토링
- 서버 20개 이상, 프론트 10개 이상의 커밋으로 대규모 코드 개선
- 코드 품질 및 유지보수성 대폭 향상

### 3. 사용자 경험 개선
- 신규 환자 정보 입력 인터렉션 개선 (태블릿, QR)
- 환자 수가 1명일 때 자동 체크 기능 수정
- 환자 목록 엑셀 업로드 기능 개선

### 4. API 성능 이슈 식별
- 의사 리스트 조회 API의 Slow Query 문제 파악 및 개선 계획 수립
