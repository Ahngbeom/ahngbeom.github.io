## 개요

2025년은 플라잉닥터 서비스의 **MSA 완성, 보안 강화, SaaS 플랫폼 확장**이 이루어진 해였습니다. 12개월간 **508개의 커밋**과 **152개의 이슈**를 처리하며, 더 안정적이고 확장 가능한 플랫폼으로 성장했습니다.

### 2025년 핵심 키워드

- **🏗️ MSA**: 결제 서비스 MSA 분리 완료
- **🔒 보안**: 개인정보 보호 강화, 토큰 기반 인증 시스템 구축
- **💼 SaaS**: 병원 홈페이지 구독 서비스 런칭

## 주요 프로젝트

### 🏗️ 아키텍처 개선

#### 결제 서비스 MSA 분리 완료 (2월)

2024년 11월에 시작한 결제 서비스 MSA 분리를 완료했습니다.

- **서비스 독립화**: 결제 로직을 완전한 독립 서비스로 분리
- **버그 대량 수정**: 분리 과정에서 발생한 29개 이슈 해결
- **장애 격리**: 결제 장애가 전체 서비스에 영향 없도록 설계 완성

**성과**: MSA 아키텍처 첫 번째 마일스톤 달성, 결제 시스템 안정성 대폭 향상

#### 진료 신청서 API 리팩토링 (10월)

기존 진료 신청서 API의 구조를 개선하여 유지보수성과 확장성을 높였습니다.

- **코드 구조 개선**: 책임 분리 및 모듈화
- **성능 최적화**: 불필요한 쿼리 제거
- **테스트 커버리지 향상**: 리팩토링 시 안정성 확보

---

### 🚀 신규 기능 개발

#### 모비닥 디지털 처방 시스템 (7~8월)

환자에게 디지털 처방을 제공하는 신규 서비스를 구축했습니다.

- 디지털 처방전 생성 및 관리
- 환자 앱 연동 처방 조회
- 병원 찾기 기능 구현

#### 포인트 충전 결제 시스템 (11월)

사용자가 포인트를 충전하여 서비스를 이용할 수 있는 결제 시스템을 구축했습니다.

- 포인트 충전 결제 프로세스
- 포인트 잔액 관리
- 결제 내역 조회

#### 병원 홈페이지 구독 서비스 SaaS (12월)

병원이 자체 홈페이지를 구독 형태로 이용할 수 있는 SaaS 플랫폼을 런칭했습니다.

- 구독 플랜 관리 시스템
- 병원 홈페이지 템플릿 제공
- 캐싱 전략 개선으로 성능 최적화

#### 외부 API 연동 (4~5월)

다양한 외부 서비스와의 연동을 통해 서비스 확장성을 높였습니다.

- 기상 정보 API 연동 (4월)
- YouTube API 연동 (5월)
- 위뉴 컨텐츠 서비스 연동 (5월)

---

### 🔧 시스템 안정화

#### 개인정보 보호 강화 (9월)

사용자 개인정보 보호를 위한 보안 체계를 강화했습니다.

- 민감 정보 암호화 강화
- 접근 권한 관리 체계 개선
- API 성능 최적화 병행

#### 토큰 기반 인증 강화 (11월)

보안 수준을 높이기 위해 토큰 인증 시스템을 강화했습니다.

- JWT 토큰 관리 체계 개선
- 토큰 만료 및 갱신 로직 최적화
- APM 모니터링 도입으로 보안 이벤트 추적

#### 힙 메모리 누수 해결 (10월)

운영 중 발견된 메모리 누수 문제를 근본적으로 해결했습니다.

- 메모리 프로파일링을 통한 원인 분석
- 누수 원인 코드 수정
- 모니터링 체계 구축으로 재발 방지

#### 가예약 시스템 안정화 (1월)

2024년 11월에 구축한 가예약 시스템의 안정화를 완료했습니다.

- 가예약 QR 접수 기능 추가 (3월)
- 환자 등록 UX 개선
- 엣지 케이스 버그 수정

## 기술적 성장

### 아키텍처 설계 경험

- **MSA 완성**: 2024년 시작한 결제 서비스 MSA 분리를 완료하며 마이크로서비스 아키텍처 경험 축적
- **SaaS 플랫폼**: 구독 기반 서비스 설계 및 구현 경험 확보
- **캐싱 전략**: 대규모 트래픽 처리를 위한 캐싱 전략 수립 및 적용

### 보안 역량 강화

- **개인정보 보호**: GDPR/개인정보보호법 준수를 위한 기술적 조치 경험
- **토큰 인증**: JWT 기반 인증 체계 설계 및 구현 노하우 축적
- **보안 모니터링**: APM을 활용한 보안 이벤트 추적 체계 구축

### 성능 최적화

- **메모리 관리**: 힙 메모리 누수 진단 및 해결 경험
- **API 최적화**: 성능 병목 분석 및 개선 노하우 확보
- **모니터링**: APM 도입으로 실시간 성능 모니터링 체계 구축

## 주요 인사이트

### 배운 점

1. **MSA 분리는 점진적으로, 완료는 확실하게**: 2024년 11월에 시작한 MSA 분리를 2025년 2월에 완료하며, 충분한 안정화 기간(29개 이슈 해결)의 중요성 체감
2. **보안은 기능이 아닌 기반**: 9월 개인정보 보호 강화와 11월 토큰 인증 강화를 통해 보안의 지속적 개선 필요성 인식
3. **APM의 가치**: 11월 APM 도입 후 메모리 누수, 성능 병목 등 운영 이슈 조기 발견 및 해결 가능
4. **SaaS는 기술만의 문제가 아님**: 12월 구독 서비스 런칭 시 기술, 비즈니스, 운영의 통합적 접근 필요성 학습

### 개선할 점

1. **프론트엔드 역량 강화**: 백엔드 중심(BE 330 vs FE 178) 작업에서 균형 잡힌 풀스택 역량 개발 필요
2. **테스트 자동화 확대**: 통합 테스트 분리(6월)를 시작했으나 전체 테스트 커버리지 확대 필요
3. **문서화**: API 문서화, 아키텍처 문서화 등 지식 공유 체계 강화 필요

## 마무리

2025년은 플라잉닥터 서비스가 **안정적이고 확장 가능한 플랫폼으로 성숙**한 해였습니다. 결제 MSA 분리 완료로 아키텍처 현대화의 첫 마일스톤을 달성하고, 개인정보 보호와 토큰 인증 강화로 보안 수준을 높였으며, 병원 홈페이지 구독 서비스 SaaS로 플랫폼 비즈니스를 확장했습니다.

2024년 대비 커밋 수가 증가(463→508)하고 이슈 수가 감소(529→152)한 것은 시스템이 안정화되면서 더 많은 기능 개발에 집중할 수 있게 되었음을 의미합니다. 2026년에는 이 기반 위에서 MSA 확대, 테스트 자동화 강화, 풀스택 역량 균형 등 더 견고한 플랫폼으로 성장할 것입니다.
