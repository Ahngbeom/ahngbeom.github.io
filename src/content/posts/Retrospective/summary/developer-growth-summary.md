## 아키텍처 진화 스토리

플라잉닥터 서비스는 3년간 **모놀리식 → MSA → 플랫폼**으로 진화했습니다. 이 과정은 단순한 기술 전환이 아니라, 비즈니스 성장에 맞춘 아키텍처 의사결정의 연속이었습니다.

### 2023년: 기반 구축 — "흩어진 것을 하나로"

2023년의 핵심 과제는 **통합**이었습니다. 방문예약과 원격진료가 별도 시스템으로 운영되던 구조를 하나의 진료 시스템으로 통합하는 대규모 리팩토링(6월, 50개 커밋)을 수행했습니다. 이 통합이 이후 바로접수(7월), 공적처방전달(12월) 같은 신규 기능 개발을 가능하게 한 토대가 되었습니다.

동시에 페이플에서 결제선생으로 결제 시스템을 전면 전환(9~12월)하면서, 기존 데이터 호환성을 유지하며 새 시스템으로 점진적 이관하는 경험을 쌓았습니다. MySQL 8 업그레이드, 테스트 코드 첫 도입, mobidoc-database 초기 구축도 이 시기에 이루어졌습니다.

> **배움**: 대규모 리팩토링은 단기 비용이 크지만, 장기적으로 신규 기능 개발 속도를 가속화한다.

### 2024년: 현대화 — "구조를 바꾸다"

2024년은 **아키텍처 현대화의 해**였습니다. 진료 테이블 v2 마이그레이션(4월, 80개+ 이슈)으로 데이터 구조를 재설계하고, 연말에는 결제 서비스 MSA 분리(11월)에 착수했습니다.

Helm Chart를 도입하여(5월~) 서비스 배포를 코드로 관리하기 시작했고, Spring REST Docs & Swagger UI(8월)로 API 문서화를 자동화했습니다. 가예약 시스템(11월), 직원관리 CRM(8월), 병원 구독 서비스(6월) 등 비즈니스 기능도 활발하게 추가되었습니다.

> **배움**: 대규모 마이그레이션은 예상보다 2배의 안정화 기간이 필요하다. MSA 전환은 반드시 점진적으로 진행해야 한다.

### 2025년: 성숙 — "플랫폼으로 도약"

2025년은 **플랫폼 성숙의 해**였습니다. 결제 MSA 분리를 완료(2월, 29개 이슈 해결)하고, Go 기반 mobidoc-pdf 서비스를 처음부터 구축(3월, 23개 커밋)하여 새로운 언어를 프로덕션에 도입했습니다. Knative를 활용한 서버리스 배포, n8n+Jenkins+GitLab CI/CD를 연결한 DDL 자동화 파이프라인(12월, 38개 커밋)을 구축하며 DevOps 역량을 크게 확장했습니다.

보안 측면에서는 개인정보 보호 강화(9월), JWT 토큰 인증(11월), APM 모니터링 도입(11월)으로 운영 수준을 높였고, 병원 홈페이지 구독 서비스 SaaS(12월)를 런칭하며 플랫폼 비즈니스를 확장했습니다.

> **배움**: 이슈 수가 529→152로 줄고 커밋이 486→700으로 늘었다는 것은, 시스템이 안정화되면서 더 많은 기능 개발에 집중할 수 있게 되었음을 의미한다.

### 2026년 (진행 중): 고도화 — "품질을 끌어올리다"

2026년 초에는 **성능 최적화와 품질 강화**에 집중하고 있습니다. mobidoc-pdf의 전역 뮤텍스를 제거하여 완전 병렬화를 달성하고, 벤치마크 테스트 60개 시나리오를 구축했습니다. Samsung Health SDK 1.0.0을 통합하며 Native(Kotlin) → Plugin → Dart → WebView 전체 레이어를 연결하는 경험을 쌓았습니다.

## 주요 인사이트 & 교훈

3년간의 경험에서 축적된 핵심 배움을 정리합니다.

### 아키텍처 의사결정

1. **대규모 리팩토링은 장기 투자다**: 2023년 6월 진료 통합(50개 커밋)이 이후 바로접수, 공적처방전달 등 신규 기능 개발을 가속화했습니다. 단기 비용을 감수한 구조적 개선이 장기적 생산성을 높였습니다.

2. **MSA 전환은 점진적으로, 완료는 확실하게**: 2024년 11월 착수 → 2025년 2월 완료까지 4개월, 29개 이슈 해결. 빅뱅 전환이 아닌 점진적 분리가 안정성을 보장했습니다.

3. **마이그레이션 안정화 기간은 예상의 2배**: 진료 테이블 v2(2024.04)에서 80개+ 이슈가 발생하며 체감한 교훈입니다.

### 기술적 성장

4. **새로운 언어는 프로덕션에서 배운다**: Go 기반 mobidoc-pdf를 처음부터 구축하며 언어를 실전에서 익혔습니다. 2026년에는 뮤텍스 세분화, 벤치마크 60개 시나리오까지 깊이가 더해졌습니다.

5. **자동화는 복리 효과를 낸다**: mobidoc-database의 DDL 자동화 파이프라인(n8n → GitLab → Jenkins)은 초기 구축에 38개 커밋이 필요했지만, 이후 스키마 변경 관리 비용을 크게 줄였습니다.

6. **테스트는 단계적으로 확장하라**: 2023년 단위 테스트 도입 → 2024년 API 문서화 → 2025년 통합 테스트 분리 → 2026년 부하 테스트/벤치마크. 한 번에 모든 테스트를 도입하기보다 단계적으로 확장하는 것이 현실적입니다.

### 운영 & 비즈니스

7. **보안은 기능이 아닌 기반이다**: 2025년 개인정보 보호(9월) + JWT 토큰(11월) + APM(11월)을 순차 도입하며 보안이 사후 대응이 아닌 지속적 개선의 대상임을 체감했습니다.

8. **SaaS는 기술만의 문제가 아니다**: 2025년 12월 병원 홈페이지 구독 서비스 런칭 시, 기술 구현뿐 아니라 비즈니스 모델, 과금 체계, 운영 프로세스까지 통합적으로 설계해야 했습니다.

## 마무리

3년이라는 시간 동안 **플라잉닥터(모비닥) 플랫폼**은 진료 예약 서비스에서 결제 시스템, 구독 SaaS, 건강 데이터 플랫폼까지 확장되었습니다. 개발자로서의 역할도 함께 성장했습니다.

- **2023년**에는 흩어진 시스템을 통합하고 결제 인프라를 전환하며 **서비스의 뼈대**를 세웠습니다.
- **2024년**에는 데이터 구조를 재설계하고 MSA 전환을 시작하며 **아키텍처를 현대화**했습니다.
- **2025년**에는 MSA를 완성하고 Go 서비스, SaaS, DDL 자동화를 도입하며 **플랫폼으로 성숙**시켰습니다.
- **2026년**에는 병렬화, 벤치마크, Health SDK 통합으로 **품질과 성능을 고도화**하고 있습니다.

약 1,874개의 커밋과 1,063개의 이슈를 처리하며 쌓은 경험의 핵심은 이것입니다:

> 좋은 소프트웨어는 한 번에 만들어지지 않는다. 통합하고, 전환하고, 분리하고, 자동화하는 반복의 과정 속에서, 시스템도 개발자도 함께 성장한다.
